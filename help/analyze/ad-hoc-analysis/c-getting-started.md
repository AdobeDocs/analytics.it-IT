# Get Started with Ad Hoc Analysis{#concept_48936BA28FAE42DB81F1B2CD4726EB17}

>[!Important]
>Il 6 agosto 2018 Adobe ha annunciato l’intenzione di terminare Ad Hoc Analysis. Non appena disponibile, verrà comunicata la data di fine del ciclo di vita. For more information, including which versions of Java will be compatible during this period, visit [https://adobe.ly/discoverworkspace](https://adobe.ly/discoverworkspace).

Potete eseguire analisi istantanea e avanzata sull'attività del sito Web. Puoi visualizzare più rapporti contemporaneamente e applicare segmenti su più dimensioni. Potete analizzare i dati sia dalle prospettive micro che dalle macro, per visualizzarne l'impatto sulle metriche aziendali importanti.

Queste funzioni consentono di rispondere alle domande sul traffico del sito, sulla demografia dei visitatori, sui ricavi e sul movimento dei prodotti. Puoi filtrare, ordinare e segmentare i dati per trovare le risposte a domande precise. I risultati vengono restituiti quasi istantaneamente, consentendo di analizzare rapidamente gli effetti di una combinazione di fattori.

## Java Upgrade Instructions {#section_E4C0C6492FF24636A0FF71A59331111D}

A partire da luglio 2018, Ad Hoc Analysis supporterà solo Java 8 o versione successiva. Se dopo la versione di manutenzione di luglio 2018 scegliete di eseguire Ad Hoc Analysis su Java 7, Adobe non fornirà più supporto per l'implementazione Ad Hoc Analysis.

I nostri file .jar sono firmati con una cifratura sicura a 256 bit non supportata dalle versioni di Java precedenti alla 1.7.0_76. Questo certificato con cifratura a 256 bit offre maggiore sicurezza.

Se avete installato Java 7, dovete effettuare l'aggiornamento prima della versione di manutenzione di luglio 2018. effettuando le seguenti operazioni:

* Se siete autorizzati a installare programmi sul computer:

   1. Andate a https://www.java.com.
   1. Fai clic su **[!UICONTROL Free Java Download]**.
   1. Fai clic su **[!UICONTROL Agree and Start Free Download]**.
   1. Installate la versione Java più recente specifica per il sistema operativo in uso.

* If you are **not** allowed to install programs on your machine:

   1. Consultate il reparto IT per ottenere la versione più recente di Java installata.

## Projects and Workspaces {#concept_FAE346335B0347A192C6C806C775D72B}

Un progetto definisce il set di dati importati, inclusa una suite di rapporti e un intervallo di date. Un progetto è costituito da un qualsiasi numero di report con tutte le loro metriche, impostazioni, dimensioni e segmenti. Puoi iniziare un nuovo progetto, caricare un progetto salvato o caricare il progetto che salva automaticamente.

<!-- 

c_projects.xml

 -->

You group reports into a *workspace*. Un progetto può contenere più aree di lavoro e un'area di lavoro può contenere più rapporti. La relazione tra questi elementi è più indicata come nidificata:

![](assets/project_workspace.png)

Potete aprire un solo progetto alla volta. Tuttavia, potete aprire più aree di lavoro in un progetto. In ciascuna area di lavoro potete aprire più rapporti.

The default date range for a new project is [!UICONTROL Last 90 Days].

## Report Templates {#concept_370F674C5B4C45368731AA801C5A45F8}

I modelli sono punti di partenza per il tipo di analisi che si desidera eseguire. Un modello può essere un canvas vuoto, ad esempio un rapporto Classifica o Abbandono. Oppure, il modello è un rapporto che inizia con metriche e dimensioni predefinite.

<!-- 

c_templates.xml

 -->

You can access templates when creating a project ( **[!UICONTROL File]** &gt; **[!UICONTROL New Project]**), or by adding a workspace or report.

| Modello | Descrizione |
|--- |--- |
| Ranked (Classificati) | Fornisce un quadro vuoto in cui è possibile creare una tabella. Ad esempio, un rapporto Pagine classifica le pagine sul sito in base al traffico. La tabella dei dettagli mostra percentuali e numeri per metriche quali Visualizzazioni pagina e Entrate. |
| Trended (Di tendenza) | Consente di analizzare il modo in cui le conversioni e gli eventi puntano alla granularità temporale selezionata (Ora, Giorno, Settimana, Mese, Trimestre o Anno) durante un periodo di reporting. |
| Totali | Report a livello di esecutivo che mostra le cifre sottostanti. Contiene dati per Entrate totali, Visualizzazioni pagina e Ordini. |
| Abbandono | Consente di creare un funnel che visualizza tassi di conversione e abbandono tra i punti di controllo. Ad esempio, puoi tenere traccia dei punti di abbandono di un visitatore durante un processo di acquisto. |
| Flusso | Mostra i percorsi più comuni seguiti dagli utenti in pagine, sezioni del sito e server. |
| Funnel di conversione | Mostra percentuali di conversione tra eventi di metrica specifici. Puoi utilizzare questo rapporto per comprendere il numero di click-through che generano vendite e il numero di unità vendute. |
| Analisi del sito | Uno strumento di percorsi del sito tridimensionale che mostra il modo in cui i visitatori si spostano tra pagine ed eventi specifici. |
| Virtual Focus Group | Visita a caso una visita dalle visite e mostra un'ampia quantità di dati relativi alla visita. Puoi usare il rapporto per creare gruppi di interesse dalla popolazione dei visitatori. |

## Open a Report {#task_0AC455CDA198497AA546622FB05F300D}

Puoi aprire un rapporto o un modello quando crei un progetto o direttamente da un progetto esistente. Utilizzate un modello per configurare un rapporto da zero.

<!-- 

t_reports_opening.xml

 -->

Sono disponibili diversi modi per aprire un report:

* On the [!UICONTROL New Report] page, search for a report or choose a template.
* Click **[!UICONTROL Reports]** from the menu, then select a report or report template.
* Launch a report from a dimension: right-click a dimension name, then select **[!UICONTROL Run Report]** &gt; **`report name`**.
